# GitHub 仓库设置建议

## 1. 仓库名称

建议保持：

- `ruoyi-modern-admin`
- 或 `shadcn-vue-ruoyi`

如果你更强调前后端一体，也可以使用：

- `ruoyi-modern-fullstack`

## 2. About 区域文案

### Description

建议使用：

`A modern RuoYi admin rebuilt with Vue 3, shadcn-vue, Tailwind CSS v4, and a NestJS + Prisma backend.`

### Website

如有在线预览地址，可填写预览站点；没有可暂时留空。

## 3. Topics 建议

建议至少添加这些 topics：

- `vue3`
- `typescript`
- `vite`
- `tailwindcss`
- `shadcn-vue`
- `nestjs`
- `prisma`
- `admin-template`
- `admin-dashboard`
- `ruoyi`
- `management-system`
- `fullstack`

## 4. 社交预览图建议

建议使用当前工作台截图作为 GitHub Social Preview，优先选择：

- `docs/screenshots/dashboard.png`

## 5. 置顶信息建议

仓库首页的可见信息建议优先保证：

- README 首屏说明项目定位
- 预览图可正常显示
- Apache-2.0 协议徽章可见
- 前后端技术栈清晰
- 快速开始命令可直接复制

## 6. Release 建议

首个 release 可命名为：

- `v1.0.0`

建议把以下文件作为 release 关联文档：

- `README.md`
- `docs/release-notes-v1.0.0.md`
- `docs/deployment-initialization-guide.md`
- `docs/deployment-runtime-guide.md`

## 7. 开源协作建议

当前仓库已具备这些基础协作文件：

- `LICENSE`
- `CONTRIBUTING.md`
- `SECURITY.md`
- `CODE_OF_CONDUCT.md`
- `PULL_REQUEST_TEMPLATE.md`
- `ISSUE_TEMPLATE/*`

发布前只需要再确认：

- 仓库描述是否已填写
- Topics 是否已添加
- Social Preview 是否已设置
- Release Notes 是否已粘贴

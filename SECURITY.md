# 安全策略

感谢你帮助改进项目安全性。

## 支持范围

当前建议优先关注以下分支或发布版本：

- `main`
- 最新发布版本

如果问题仅存在于历史分支、个人实验分支或已废弃实现中，修复优先级会相对降低。

## 漏洞反馈方式

如果你发现了安全问题，请不要直接提交公开 issue。

建议按以下方式处理：

1. 优先通过仓库维护者提供的私有联系方式反馈
2. 清楚说明影响范围、复现步骤、利用条件和潜在风险
3. 如有可行的缓解方案，也请一并说明

如果仓库所有者后续提供了专用安全邮箱或私有报告渠道，请将本文件中的联系方式补充完整。

## 建议包含的信息

请尽量提供：

- 漏洞类型
- 受影响模块或接口
- 复现步骤
- 影响范围
- 是否需要认证
- 是否可能导致数据泄露、权限提升、远程执行或文件写入
- 相关日志、截图或 PoC

## 披露原则

- 在问题确认并完成修复前，请避免公开披露细节
- 修复完成后，可以在维护者确认的前提下公开披露
- 如问题影响第三方依赖，请同时参考对应依赖的安全披露流程

## 当前重点关注范围

本仓库当前更值得优先检查的区域包括：

- 认证与权限链路
- 动态路由和隐藏页访问控制
- 文件上传与静态访问
- 导出与下载接口
- 后端 Prisma 写入链路
- 代码生成、富文本和头像上传等高交互模块

## 结论

负责任的安全披露会比公开提交复现细节更有价值。感谢你的协助。
